# Exploration — persona-canonical-channel

Stop paying the persona text twice per session: today the Gentleman/Neutral persona is injected BOTH as a system-prompt section AND as an output style for Claude Code — and, discovered here, for Kimi too.

## Current State

Persona injection: `internal/components/persona/inject.go`. `personaContent()` (:495) resolves a per-agent markdown asset written via one of 6 `SystemPromptStrategy` values. Blocks 3/3a (:345-397) write the Claude Code output-style file and merge `outputStyle` into settings.json, gated by `adapter.SupportsOutputStyles()`.

**Duplication surface (corrects the prior audit):**
- Only `claude/adapter.go` returns true for `SupportsOutputStyles()` — but that flag does NOT capture the full duplication.
- **Kimi also duplicates**: `internal/assets/kimi/KIMI.md:1-3` unconditionally includes BOTH `persona.md` and `output-style.md` as Jinja modules; `injectInternal`'s `StrategyJinjaModules` branch (inject.go:269-309) writes both files regardless of the capability flag. `docs/agents.md:9-26` says "Output Styles: No" for Kimi — accurate for the settings.json mechanism, misleading about the duplication.
- **The Claude assets have DRIFTED**: `claude/persona-gentleman.md` vs `claude/output-style-gentleman.md` are independently-authored paraphrases of the same concepts, not copies. Only "Persona Scope" is near-verbatim (persona.md has 3 extra "For those artifacts" bullets, :32-35). Canonicalizing fixes active drift, not just token cost.

**What must SURVIVE in the system-prompt file** (no output-style equivalent; action/tooling directives, not tone): `## Rules` (incl. Claude-CLI-specific tool bullets like "never use cat/grep/find/sed/ls"), `## Expertise`, `## Contextual Skill Loading (MANDATORY)`. Verified `keep-coding-instructions: true` (`output-style-gentleman.md:4`, `inject_test.go:192-193`, `e2e_test.sh:557`) — CLAUDE.md keeps loading alongside the style every session.

**Output-style mechanism verified**: settings.json `outputStyle` + `~/.claude/output-styles/*.md`; canonical-channel decision can be STATIC per adapter (`SupportsOutputStyles()` for Claude; explicit carve-out needed for Kimi) — no runtime detection required.

**Spec compatibility**: `openspec/specs/persona-behavior-contract/spec.md` already mandates the output-style channel be meaningful ("Claude Neutral Output Style Contract", "Kimi Neutral Output Style Content") — no conflict with making it canonical.

**Idempotency/migration**: `InjectMarkdownSection` does marker-delimited full replace (`filemerge/section.go:265`) — existing users' full persona sections converge to slim on re-inject, no orphan risk. Legacy fingerprints (`["## Personality","Senior Architect","## Rules"]` in `uninstall/cleaners.go:18-22` and `filemerge/section.go:17-21`) inspect on-disk OLD content — safe for this migration; re-validate if a future design removes those literals from all assets.
